# TenderGen - Enterprise-Grade DOCX Generator

**TenderGen** -это Local-First веб-приложение для детерминированного автозаполнения тендерных документов (DOCX) на основе структурированных JSON-данных. 

Система спроектирована с упором на строгую конфиденциальность (Zero-Backend): вся обработка файлов, маппинг данных и компиляция ZIP-архивов происходят исключительно в оперативной памяти браузера клиента. Никакие данные не покидают контур устройства.

## 🏗 Ключевые архитектурные решения

- **XML-Препроцессор "на лету":** Приложению не нужны заранее подготовленные технические шаблоны для таблиц. Ядро содержит XML-парсер, который распаковывает сырой `.docx` архив, находит таблицу (Спецификацию) и программно патчит XML-теги (`word/document.xml`), внедряя маркеры цикла `[#items]` без ручной разметки исходников.
- **Изоляция бизнес-логики:** Правила заполнения полностью отвязаны от JS-кода и вынесены во внешний конфигурационный файл `mapping.json`. Добавление новых типов документов не требует пересборки проекта.
- **Safe Data Binding:** Внедрен механизм безопасного извлечения данных с защитой от пустот. При отсутствии ключа в JSON движок не ломается и не выводит артефакты (встроен `nullGetter` для подавления `undefined`), оставляя корректные пустые поля.
- **Tech-UI / Cyberpunk:** Визуальный слой построен на Tailwind CSS v4 с использованием эффектов стекла (backdrop-blur) и кастомной геометрии без тяжелых UI-библиотек вроде MUI/AntD.

## 🛠 Технический стек

- **Core:** React 19, Vite
- **State Management:** Zustand (управление файлами и логами терминала)
- **Document Engine:** `docxtemplater`, `pizzip` (XML-манипуляции и рендер)
- **File Processing:** `jszip`, `file-saver`
- **Styling:** Tailwind CSS v4, Lucide React

⚙️ Как тестировать систему
Все необходимые тестовые данные находятся в папке Demo_Data в корне репозитория.

Загрузка данных: Выделите все 7 файлов из папки Demo_Data (3 сырых шаблона .docx, 3 файла с данными *_example.json и 1 конфиг mapping.json) и перетащите их в зону Drag & Drop в приложении.
Примечание: Система автоматически нормализует имена файлов под капотом (отрезает постфиксы _example), поэтому переименовывать исходники не нужно.

Компиляция: Нажмите кнопку «Сгенерировать пакет».

Мониторинг: Процесс XML-патчинга и сборки будет детально транслироваться во встроенном терминале.

Результат: Браузер автоматически скачает файл Tender_Documents.zip с готовыми документами.

🗂 Структура маппинга (mapping.json)
Логика связывания тегов из DOCX (в квадратных скобках [ ]) с путями в загруженных JSON-файлах имеет строгую иерархию, что позволяет масштабировать генератор на десятки документов.

Пример структуры:
JSON
{
  "docs": {
    "anketa": {
      "template_name": "01_Анкета_участника_шаблон.docx",
      "output_name": "01_Анкета_участника.docx",
      "fields": {
        "Полное наименование участника": "company_profile.company.full_name",
        "ИНН": "company_profile.company.inn"
      }
    },
    "predlozhenie": {
      "template_name": "03_Предложение_о_цене_договора_шаблон.docx",
      "output_name": "03_Предложение_о_цене.docx",
      "fields": {
        "Итого без НДС": "calc.subtotal_wo_vat"
      },
      "ARRAY:items": "calc.items"
    }
  }
}
Ключ ARRAY: инструктирует парсер извлечь массив данных для динамического развертывания строк таблиц в документах.
