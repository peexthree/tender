# TenderGen - Automated DOCX Generation

**TenderGen** — это веб-приложение для автоматизированной генерации тендерных документов (DOCX) на основе данных в формате JSON. Приложение работает исключительно на стороне клиента, обеспечивая максимальную конфиденциальность и безопасность данных: никакие документы и данные не отправляются на бэкенд.

## Особенности архитектуры

- **Локальная обработка (Client-Side Only):** Чтение файлов через File API, рендер шаблонов и создание ZIP-архива происходит прямо в браузере.
- **Поддержка динамических списков:** Механизм маппинга поддерживает прокидывание массивов из JSON (например, смет или расчетов) напрямую в `docxtemplater` для отрисовки динамических таблиц в формате `{#items} ... {/items}`.
- **Современный UI:** Использован богатый технологичный стиль, сочетающий эффекты "стекла" (glassmorphism), неон, строгую геометрию и терминал-стиль для логов, реализованный исключительно с помощью Tailwind CSS (версия 4).
- **Готовность к интеграции (TWA):** Базовый API-клиент (`src/api/client.js`) настроен с перехватчиками (interceptors) Axios и имеет заглушку для интеграции с Telegram Web Apps SDK (передача `initData`).

## Используемые технологии

- **React 19** + **Vite**
- **Tailwind CSS v4** (кастомные стили, глитч-эффекты)
- **Zustand** (глобальный стор для загруженных файлов и логов терминала)
- **docxtemplater** + **pizzip** (рендер DOCX шаблонов)
- **jszip** + **file-saver** (сборка и сохранение ZIP-архивов)
- **React Query** + **Axios** (API-интеграция)
- **Lucide React** (иконки)

## Инструкция по запуску

Убедитесь, что у вас установлен Node.js (рекомендуется v18+).

1. Установите зависимости проекта:
   `npm install`

2. Запустите сервер для разработки:
   `npm run dev ` (без пробела)

3. Откройте приложение в браузере по указанному в консоли адресу (обычно `http://localhost:5173`).

4. Для сборки production-версии:
   `npm run build`

## Как использовать приложение

1. **Загрузка файлов:**
   В зоне `Drag & Drop` перетащите необходимые файлы:
   - Файлы данных: `company_profile.json`, `calc.json`, сгенерированный `tender.json` и т.д.
   - Файл маппинга: `mapping.json`.
   - Шаблоны документов: `*.docx`.

2. **Генерация:**
   Нажмите кнопку "Сгенерировать". Процесс генерации будет логироваться в панели "Terminal Output" справа.
   По завершении автоматически начнется скачивание архива `Tender_Documents.zip` со всеми заполненными документами.

## Структура маппинга (`mapping.json`)

Маппинг позволяет связать теги из DOCX шаблона с путями в загруженных JSON-файлах.

Пример `mapping.json`:
```json
{
  "Анкета.docx": {
    "company_name": "company_profile.name",
    "customer": "tender.customer.full_name",
    "ARRAY:items": "calc.items"
  },
  "default": {
    "date": "tender.bid_deadline"
  }
}
```
*Примечание: Префикс `ARRAY:` используется для явного указания, что по данному пути нужно извлечь массив (например, для таблиц в DOCX).*
